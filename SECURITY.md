# 安全政策

## 支持的版本

目前仅支持最新版本的安全更新。

| 版本 | 支持状态 |
| ---- | ------ |
| 1.0.x | ✅ 支持 |
| < 1.0 | ❌ 不支持 |

## 报告安全漏洞

如果您发现安全漏洞，请**不要**通过公开 Issue 报告。

请通过以下方式私下报告：

1. 发送邮件至项目维护者
2. 使用 GitHub Security Advisories
3. 在 Issue 中简单描述问题，等待维护者联系

### 报告内容

请包含以下信息：

- 漏洞类型（如：XSS、SQL 注入等）
- 影响的版本
- 复现步骤
- 可能的修复建议

### 响应时间

- 确认收到报告：24 小时内
- 初步评估：3 天内
- 修复时间：视严重程度而定

## 安全最佳实践

### API 密钥管理

- **不要**将 API 密钥提交到版本控制
- 使用 `.env` 文件存储敏感配置
- 使用 Supabase RLS 策略保护数据

### 用户数据

- 任务数据存储在用户本地设备
- 照片上传到 Supabase Storage，仅分享链接可访问
- 支持匿名使用，无需注册账号

## 已知安全考虑

1. **匿名认证**：应用使用 Supabase 匿名认证，无需用户提供个人信息
2. **本地存储**：敏感数据仅存储在用户设备
3. **分享链接**：拍照分享链接不包含身份信息
